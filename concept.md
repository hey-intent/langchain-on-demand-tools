# Core Concepts

This document explains the fundamental concepts behind AI agents and multi-agent systems.

---

## Agent

An **agent** is an LLM enhanced with the ability to take actions through **tools**.

### Components

```
┌─────────────────────────────────────────────────────────────────┐
│                            AGENT                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                         LLM                             │   │
│   │         (Language Model - the "brain")                  │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              │ can invoke                       │
│                              ▼                                  │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                        TOOLS                            │   │
│   │  ┌───────────┐  ┌───────────┐  ┌───────────┐            │   │
│   │  │  Tool A   │  │  Tool B   │  │  Tool C   │   ...      │   │
│   │  └───────────┘  └───────────┘  └───────────┘            │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Agent Execution Flow

```
┌──────────┐          ┌──────────┐          ┌──────────┐          ┌──────────┐
│          │          │          │          │          │          │          │
│   USER   │          │   LLM    │          │ RUNTIME  │          │   TOOL   │
│          │          │          │          │          │          │          │
└────┬─────┘          └────┬─────┘          └────┬─────┘          └────┬─────┘
     │                     │                     │                     │
     │  1. "What is 2+2?"  │                     │                     │
     │────────────────────>│                     │                     │
     │                     │                     │                     │
     │                     │  2. Tool call:      │                     │
     │                     │     calculate(2+2)  │                     │
     │                     │────────────────────>│                     │
     │                     │                     │                     │
     │                     │                     │  3. Execute tool    │
     │                     │                     │────────────────────>│
     │                     │                     │                     │
     │                     │                     │  4. Result: 4       │
     │                     │                     │<────────────────────│
     │                     │                     │                     │
     │                     │  5. Tool result: 4  │                     │
     │                     │<────────────────────│                     │
     │                     │                     │                     │
     │  6. "2+2 equals 4"  │                     │                     │
     │<────────────────────│                     │                     │
     │                     │                     │                     │
     ▼                     ▼                     ▼                     ▼
```

## Tool

A **tool** is a function that an agent can invoke to interact with the outside world.

### Components

```
┌─────────────────────────────────────────────────────────────────┐
│                            TOOL                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. DEFINITION (Schema)                                        │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  • name: "calculate"                                    │   │
│   │  • description: "Evaluate a math expression"            │   │
│   │  • parameters: { expression: string }                   │   │
│   │                                                         │   │
│   │  → This is what the LLM sees and understands            │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              +                                  │
│   2. IMPLEMENTATION (Code)                                      │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │                                                         │   │
│   │  → This is what actually runs when the tool is called   │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Skill

A **skill** is a reusable capability that can be injected into an agent to extend its behavior without modifying the agent's core logic.

### Components

```
┌─────────────────────────────────────────────────────────────────┐
│                            SKILL                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. METADATA                                                   │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  • name: "web-search"                                   │   │
│   │  • description: "Search the web for information"        │   │
│   │  • version: "1.0.0"                                     │   │
│   │  • tags: ["search", "web", "research"]                  │   │
│   │                                                         │   │
│   │  → Identifies and describes the skill                   │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              +                                  │
│   2. TOOL(S)                                                    │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  ┌───────────┐  ┌───────────┐                           │   │
│   │  │  Tool A   │  │  Tool B   │   ...                     │   │
│   │  └───────────┘  └───────────┘                           │   │
│   │                                                         │   │
│   │  → The actual capabilities the skill provides           │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Skill vs Tool

```
┌─────────────────────────────────────────────────────────────────┐
│                      SKILL vs TOOL                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   TOOL (Low-level)                                              │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  • Single function with defined inputs/outputs          │   │
│   │  • No behavioral context                                │   │
│   │  • Example: "fetch_url(url) -> content"                 │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   SKILL (High-level)                                            │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  • Bundles tools + metadata                             │   │
│   │  • Tool descriptions provide behavioral context         │   │
│   │  • Example: "web-search" skill that knows WHEN and      │   │
│   │    HOW to search via well-crafted tool descriptions     │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   Analogy:                                                      │
│   • Tool = A hammer                                             │
│   • Skill = Carpentry (knows when to use hammer, nails, etc.)   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Skill Injection Flow

```
┌──────────┐          ┌──────────┐          ┌──────────┐          ┌──────────┐
│          │          │          │          │          │          │          │
│  SKILL   │          │ RUNTIME  │          │  AGENT   │          │   LLM    │
│ REGISTRY │          │          │          │          │          │          │
└────┬─────┘          └────┬─────┘          └────┬─────┘          └────┬─────┘
     │                     │                     │                     │
     │  1. Register skill  │                     │                     │
     │<────────────────────│                     │                     │
     │                     │                     │                     │
     │  2. Load skills     │                     │                     │
     │     for agent       │                     │                     │
     │────────────────────>│                     │                     │
     │                     │                     │                     │
     │                     │  3. Inject:         │                     │
     │                     │     - Tools         │                     │
     │                     │────────────────────>│                     │
     │                     │                     │                     │
     │                     │                     │  4. Enhanced agent  │
     │                     │                     │     with skills     │
     │                     │                     │────────────────────>│
     │                     │                     │                     │
     │                     │                     │  5. LLM can now     │
     │                     │                     │     use skill tools │
     │                     │                     │     with context    │
     │                     │                     │<────────────────────│
     │                     │                     │                     │
     ▼                     ▼                     ▼                     ▼
```

## Multi-Agent

A **multi-agent system** involves multiple agents coordinated by a runtime/orchestrator.

### Multi-Agent Execution Flow

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│          │    │          │    │          │    │          │    │          │
│   USER   │    │ RUNTIME  │    │ AGENT 1  │    │ AGENT 2  │    │  HUMAN   │
│          │    │(Orchestra│    │ (Router) │    │ (Worker) │    │  INPUT   │
│          │    │  tor)    │    │          │    │          │    │          │
└────┬─────┘    └────┬─────┘    └────┬─────┘    └────┬─────┘    └────┬─────┘
     │               │               │               │               │
     │ 1. Request    │               │               │               │
     │──────────────>│               │               │               │
     │               │               │               │               │
     │               │ 2. Analyze    │               │               │
     │               │──────────────>│               │               │
     │               │               │               │               │
     │               │ 3. Route info │               │               │
     │               │<──────────────│               │               │
     │               │               │               │               │
     │               │ 4. Need human │               │               │
     │               │   approval?   │               │               │
     │               │──────────────────────────────────────────────>│
     │               │               │               │               │
     │               │ 5. Approval   │               │               │
     │               │<──────────────────────────────────────────────│
     │               │               │               │               │
     │               │ 6. Delegate   │               │               │
     │               │   task        │               │               │
     │               │──────────────────────────────>│               │
     │               │               │               │               │
     │               │               │               │ 7. Process    │
     │               │               │               │    (may use   │
     │               │               │               │     tools)    │
     │               │               │               │               │
     │               │ 8. Result     │               │               │
     │               │<──────────────────────────────│               │
     │               │               │               │               │
     │ 9. Response   │               │               │               │
     │<──────────────│               │               │               │
     │               │               │               │               │
     ▼               ▼               ▼               ▼               ▼
```

### Multi-Agent Patterns

```
┌─────────────────────────────────────────────────────────────────┐
│                    ──────────────────────┘    │
│                                          MULTI-AGENT PATTERNS                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. ROUTER PATTERN (this project)                               │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │   User ──> Router Agent ──> Orchestrator ──> Worker     │    │
│  │                                              Agent      │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
│  2. PIPELINE PATTERN                                            │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │   User ──> Agent A ──> Agent B ──> Agent C ──> Result   │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
│  3. SUPERVISOR PATTERN                                          │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │              ┌──> Agent A ──┐                           │    │
│  │   User ──> Supervisor ──> Agent B ──> Supervisor ──>    │    │
│  │              └──> Agent C ──┘              Result       │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
│  4. COLLABORATIVE PATTERN                                       │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │   Agent A <───────> Agent B                             │    │
│  │      ▲                 ▲                                │    │
│  │      │                 │                                │    │
│  │      └───── Agent C ───┘                                │    │
│  └───────────────────────────────────                        │
└─────────────────────────────────────────────────────────────────┘
```
